# Codex Project Notes

These notes are for future Codex work in this repo. They summarize the project structure, how the site works, and the preferred blog/SEO style.

## Project Overview

- Site: `Eks Emigrantai` / `https://exemigrantai.lt`.
- Purpose: Lithuanian-language practical guidance for people moving between the UK and Lithuania, especially documents, benefits, pensions, employment history, relocation, customs, and everyday admin.
- Stack: Astro 5, Astro content collections, Markdoc posts, Keystatic Cloud, Tailwind CSS 4, Cloudflare adapter, sitemap, RSS.
- Runtime shape: static output (`output: 'static'`) deployed through Cloudflare Pages.
- CMS: Keystatic Cloud project `solid-digital/exemigrantai-blog`.

## Important Commands

Run from the repo root:

```sh
npm install
npm run dev
npm run build
npm run preview
npm run import:wordpress
npm run normalize:posts
```

Notes:

- `npm run dev` starts Astro locally.
- `npm run build` validates Astro content schemas and generates the static site.
- `npm run import:wordpress` is for migration/import work.
- `npm run normalize:posts` normalizes post frontmatter.

## Key Files And Folders

- `astro.config.mjs`: Astro config. Uses Cloudflare adapter, Markdoc, React, Keystatic, sitemap, and Tailwind Vite plugin. Site URL is `https://exemigrantai.lt`.
- `keystatic.config.ts`: Keystatic schema and CMS config. Posts live under `src/content/posts/*/`; images use `src/assets/images/posts` and public path `@assets/images/posts/`.
- `src/content.config.ts`: Astro content collection schemas for `posts` and `settings`.
- `src/consts.ts`: Global site constants: `SITE_TITLE`, `SITE_ALT_NAME`, `SITE_URL`, `SITE_DESCRIPTION`.
- `src/lib/content.ts`: Content helpers: post sorting, pagination, date parsing, reading time, generated excerpts, settings fallback.
- `src/content/posts/*/index.mdoc`: Blog posts. Each post is a folder named by slug with an `index.mdoc`.
- `src/assets/images/posts/*`: Featured images and embedded post images.
- `src/content/settings/index.yaml`: Homepage title, navigation, Facebook group URL, footer text.
- `src/pages/index.astro`: Homepage. Shows hero, featured latest post, remaining post grid, pagination.
- `src/pages/blog/[slug].astro`: Blog article page. Renders Markdoc content, article SEO, structured data, reading time, and view counter.
- `src/pages/page/[page].astro`: Paginated archive pages from page 2 onward.
- `src/pages/rss.xml.js`: RSS feed generated from posts.
- `src/components/BaseHead.astro`: Shared SEO metadata, canonical URL, Open Graph, Twitter cards, favicon, sitemap/RSS links.
- `src/components/StructuredData.astro`: JSON-LD renderer.
- `src/components/PostCard.astro`: Post grid card.
- `src/components/Header.astro`, `Footer.astro`, `ScrollToTop.astro`: Site chrome and small client behavior.
- `src/styles/global.css`: Global theme, layout, prose styles, responsive behavior.
- `public/robots.txt`, `public/favicon.svg`: Static public assets.
- `wrangler.pages.example.jsonc`: Non-active Cloudflare Pages config example.

Avoid editing generated/cache folders unless specifically requested:

- `.astro`
- `.wrangler`
- `dist`
- `node_modules`

## Content Model

Posts are Astro content entries with this frontmatter:

```yaml
---
title: "Post title"
publishedAt: "2026-03-27T11:15:00.000Z"
category: "Naujienos"
featuredImage: "@assets/images/posts/post-slug/featuredImage.png"
featuredImageAlt: "Descriptive image alt text"
seoTitle: "Optional shorter SEO title"
seoDescription: "Optional SEO meta description"
---
```

Required:

- `title`
- `publishedAt`

Optional:

- `category`, defaults to `Naujienos`
- `featuredImage`
- `featuredImageAlt`
- `seoTitle`
- `seoDescription`

Dates may be strings or Date objects. `parsePublishedAt()` treats strings without timezone as UTC by appending `Z`.

## Routing And Rendering

- Homepage route: `/`
- Article route: `/blog/{slug}/`
- Archive pagination: `/page/2`, `/page/3`, etc.
- RSS: `/rss.xml`
- Sitemap is generated by `@astrojs/sitemap`.

Posts are sorted newest first in `src/lib/content.ts`.

Homepage:

- Fetches first page of posts via `getPaginatedPosts(1)`.
- Treats the first post as the featured story.
- Displays the rest as `PostCard` items.

Article pages:

- `getStaticPaths()` returns every post slug.
- `post.render()` renders Markdoc content.
- `getPostExcerpt()` generates a fallback summary from the body.
- `estimateReadingTime()` calculates approximate reading time at 200 words/minute.
- Counter API increments views using `https://api.counterapi.dev/v1/exemigrantai/{slug}/up`.

## SEO Behavior

`BaseHead.astro` provides:

- `<title>`
- meta title and description
- robots meta, default `index,follow`
- canonical URL from `Astro.url.pathname` and `Astro.site`
- Open Graph tags
- Twitter card tags
- RSS and sitemap links
- Google AdSense account meta

Article SEO:

- SEO title = `post.data.seoTitle || post.data.title`.
- SEO description = `post.data.seoDescription || generated excerpt`.
- OG/Twitter image = featured image when present, otherwise default from `BaseHead`.
- Adds `article:published_time`.
- Adds JSON-LD `BlogPosting` with headline, description, datePublished, URL, language `lt`, category, image, and publisher.

Homepage structured data:

- `WebSite`
- `Organization`
- Uses Facebook group URL in `sameAs` when available.

SEO writing expectations:

- Prefer clear, specific Lithuanian SEO titles.
- Keep `seoTitle` tighter than the visible title when the visible title is long.
- Write `seoDescription` as a direct benefit/summary, usually around 140-160 characters when practical.
- Include the main topic phrase naturally in title, intro, at least one heading, and conclusion.
- Use official source links in the body for trust, especially GOV.UK, Sodra, UZT, Lithuanian customs, or other relevant institutions.
- For legal/benefits/tax/current-rule topics, verify current official sources before writing or updating.

## Blog Writing Style

Language:

- Lithuanian.
- Plain, practical, calm, and direct.
- Written for real emigrants trying to understand what to do next, not for specialists.

Voice:

- Helpful and grounded.
- Avoid hype and overpromising.
- Explain complicated admin topics in simple steps.
- Use phrases like `Trumpas atsakymas`, `Praktikoje tai reiškia`, `Svarbi detalė`, `Čia dažnai žmonės pasimeta`.
- Be careful with official claims. Attribute rules to sources and include source links.

Typical article structure:

1. Bold opening paragraph that states the problem and practical angle.
2. Short answer or framing paragraph.
3. Source/date sentence, for example: `Šis straipsnis parengtas remiantis oficialiais ... šaltiniais, patikrintais 2026 m. kovo 27 d.`
4. H2 sections phrased as reader questions or concrete tasks.
5. Bulleted checklists for required documents, steps, common mistakes.
6. A table-style quick reference using Markdoc table syntax when useful.
7. `Ką svarbiausia prisiminti?` or similar final summary.
8. `Oficialūs šaltiniai`.
9. `Svarbi pastaba` disclaimer for legal, benefits, money, or immigration-adjacent topics.

Common headings:

- `Kas yra ...?`
- `Kada ... tikrai svarbu?`
- `Kaip gauti ...?`
- `Ką daryti, jei ...?`
- `Kokius dokumentus verta turėti?`
- `Dažniausios klaidos`
- `Greita atmintinė`
- `Ką svarbiausia prisiminti?`
- `Oficialūs šaltiniai`
- `Svarbi pastaba`

Formatting style:

- Use `##` for major sections.
- Use `###` for numbered mistake sections or subtopics.
- Use bullets generously for practical lists.
- Use numbered lists for step-by-step flows.
- Use inline code formatting for document/form names like `P45`, `P60`, `PDU1`, `SED U002`, `assessment period`.
- Avoid excessive horizontal rules unless preserving imported legacy structure.
- Markdoc table syntax is used in posts:

```mdoc
{% table %}
- Column A
- Column B
---
- Row A
- Row B
{% /table %}
```

## Current Article Patterns

Strong newer articles include:

- `src/content/posts/u1-forma-griztant-i-lietuva-is-jk/index.mdoc`
- `src/content/posts/universal-credit-ir-isvykimas-is-jk-ka-butina-sutvarkyti-pries-griztant-i-lietuva/index.mdoc`

These are good examples for:

- practical intro
- official source grounding
- cautious interpretation
- common mistakes
- quick reference tables
- disclaimer section

Some older/imported posts are more generic SEO style, with broader claims and less source precision. When updating or creating content, follow the newer practical-source-based style.

## Deployment Notes

- README says the repo is safe to deploy through the existing Cloudflare Pages dashboard setup.
- Do not hand-write an active `wrangler.jsonc` unless intentionally taking over dashboard config.
- If needed, download existing Pages config first:

```sh
npx wrangler pages download config emigrantai-blog
```

- The Cloudflare adapter may expect a KV binding named `SESSION` if session storage errors appear.

## Gotchas

- Use `Get-Content -Encoding utf8` in PowerShell when reading Lithuanian text; default output may show mojibake.
- Paths like `src/pages/blog/[slug].astro` and `src/pages/page/[page].astro` need `-LiteralPath` in PowerShell because brackets are wildcard syntax.
- `rg` may fail in this environment with `Access is denied`; fall back to PowerShell recursive listing if needed.
- Do not edit `dist` for source changes.
- Keep unrelated generated files out of manual edits.

## Recommended Workflow For Future Changes

1. Check `git status --short --branch`.
2. Read relevant source files and content examples before editing.
3. For new posts, create a slug folder under `src/content/posts/` with `index.mdoc` and place images under matching `src/assets/images/posts/{slug}/`.
4. Add complete frontmatter, including SEO fields and image alt text when possible.
5. Verify with `npm run build`.
6. For frontend changes, run the dev server and inspect responsive pages if practical.
